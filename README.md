# Go Secure Coding Practice

보안 코딩 연습 위한 프로젝트

기능 구현 부분을 모두 성공한게 아니므로 보안 관련 코드는 완성하지 못했습니다.
하지만 흐름을 이해하고 어떤 기능부터 구현하고 어떤 기준으로 나눌지 분리 기준을 생각해보며 진행했습니다

## 현재 상태

- 배포 때의 상태를 유지하고 있으며, 구현 기능만 몇가지 추가되었습니다.

## 기본 계정
- `alice / alice1234`
- `bob / bob1234`
- `charlie / charlie1234`
- `test` / `test`

## 실행 방법

프로젝트 루트에서 실행합니다

```powershell
go run ./cmd/server
```
코드 실행이 완료 됐다면 브라우저에 http://localhost:8080 을 입력해 접속합니다. 

- 처음 상태로 다시 시작하고 싶으면 `app.db`를 지운 뒤 다시 실행하면 됩니다.

## 구현 기능
- 회원가입 로직을 만들어 새로운 유저의 회원가입이 가능합니다.
- 회원가입 및 로그인을 완료한 유저는 뱅크 란에서 입금, 출금이 가능합니다.
- 로그인된 유저는 또한 게시글 작성이 가능하며 본인이 작성한 게시글 외에도 다른 유저가 작성한 게시글은 모두 게시글 목록에서 확인 가능 합니다.

## 코드
main.go
line 134 : 회원가입을 위해 insert 쿼리와 실패 시 메시지 출력 기능을 넣었습니다.

line 215 : 탈퇴 기능을 위해 Delete 쿼리와 실패 시 메시지 출력 기능, 메인으로 돌아가는 redirect 코드를 넣었습니다.

line 266 : 입금 기능을 위해 update 쿼리로 balance에 입력 금액을 더했으며 실패 시 메시지 출력, 성공 시 콘솔에 성공 메시지와 유저 이름이 나오도록 했습니다.

line 298 : 출금 기능을 위해 update 쿼리로 balance에 입력 금액을 빼고 실패 시 메시지 출력, 성공 시 콘솔에 성공 메시지와 유저 이름이 나오도록 했습니다

line 325 : 송금 기능을 위해 update 쿼리를 두번 써 targetId엔 마이너스, userid엔 플러스하며, 실패 시 각 입,출금 실패 메시지 출력, 성공 시 송금 성공 메시지가 나오도록 했습니다

line 364 : post 조회 기능을 위해 Select 문으로 게시글 목록에 나오도록 했습니다.

line 396 : post 작성 기능을 위해 Insert 쿼리를 통해 타이틀, 작성글 유저 아이디 업로드 시간, 수정 시간을 넣고 실패 시 메시지 출력과 성공시 콘솔에 ID부터 업데이트 시간까지 쭉 나오도록 했습니다. 

line 497 : 게시글 삭제를 위해 Delete 문을 적었지만 삭제 프롬프트만 나오고 실제로 삭제되지 않습니다

## +)
폴더는 총 main.go, handler, models 로 분리하려고 했습니다.
models.go 엔 데이터 구조체 정의, handler.go 엔 login logout, register 등의 기능들을 func 으로 분리해 수정 부분을 줄이고 결합도 감소를 목표로 했습니다.

Password 부분 또한 평문 저장이 아닌 bcrypt를 활용한 단방향 해싱을 적용해 데이터베이스 유출 시에도 원본 비밀번호를 보호할 수있도록 설계하고 싶었습니다.

생각보다 기능 구현에서 오래걸려 보안 관련 조치를 아무것도 하지 못한 것이 너무 아쉬웠습니다.
